# 🎤 AI音声メモ → Obsidian（Android Chrome 専用 PWA）

ワンタップで録音 → Web Speech APIでテキスト化 → Claude（軽量 `claude-haiku-4-5`）が読みやすい
日本語に清書（**個人辞書で誤変換を防止**）→ 画面表示＆**自動でクリップボードにコピー＆Obsidianへ新規ノート保存**。
さらに **Typeless風の再加工ボタン**（短く／長く／フォーマル／カジュアル／箇条書き／翻訳）付き。

ビルド不要・バックエンド不要。素のHTML/CSS/JSだけ。**GitHub Pagesに置くだけ**で公開できます。

> 公開済みの例: https://nwakao-sys.github.io/voice-memo/

## ✅ 事前に用意するもの

1. **GitHubアカウント**（無料）… https://github.com/signup
2. **Anthropic APIキー**… https://console.anthropic.com/ → 「API Keys」で発行
   - ⚠️ キーは**端末内に保存され、ブラウザから直接APIへ送信**されます。必ず**利用上限／予算を
     絞った専用キー**を作ってください（末尾の注意書き参照）。
3. **Android版 Obsidian**（保存機能を使う場合）… Google Play からインストールし、
   **保存先にする保管庫（vault）を作成して開いておく**。
   - 設定の **vault名は、Obsidianの保管庫名と完全一致**させる必要があります（既定 `あとづけ屋_Wiki_Raw`）。

## 🚀 メイン手順：GitHub Pages で公開する

### ステップ1. リポジトリを作る
1. https://github.com/new を開く
2. **Repository name** に `voice-memo`
3. **Public** を選択 → **Create repository**

### ステップ2. ファイルをアップロード
1. **Add file → Upload files**
2. `voice-memo` の中身を**まるごとドラッグ＆ドロップ**（`icons` フォルダも構成ごと）
3. **Commit changes**

### ステップ3. Pages を有効化
1. **Settings → Pages**
2. **Source** = Deploy from a branch
3. **Branch** = `main` ／ フォルダ `/(root)` → **Save**
4. 1〜2分で公開URLが表示（例 `https://ユーザー名.github.io/voice-memo/`）

### ステップ4. スマホで開いて設定
1. **Android Chrome** で公開URLを開く（HTTPSなのでマイク・コピー・Obsidian起動が動作）
2. 右上 ⚙️ で設定を開く：
   - **Anthropic APIキー**（必須）
   - モデル（空欄でOK＝`claude-haiku-4-5`）
   - **Obsidian**: vault名／フォルダ（既定 `音声メモ`）／ファイル名テンプレ（既定 `YYYY-MM-DD_HHmm`）
   - **自動Obsidian保存**トグル（既定ON）
   - **個人辞書**: 固有名詞・専門用語を改行で（例 あとづけ屋／マルチナビ／CANバス）
3. **保存してメインへ**

### ステップ5. ホーム画面に追加（アプリ化）
- 初回に下部の **「ホーム画面に追加」** をタップ（出ない場合は Chrome ⋮ → アプリをインストール）

## 🟢 使い方

1. 中央の丸ボタンをタップ → **録音開始**（赤く点滅）
2. 話す（連続録音。途中で切れても自動再開）
3. もう一度タップ → **停止** → AIが清書 → 結果表示
4. **自動でコピー**され、**自動保存ONならObsidianが開いて新規ノート作成**
5. 清書の下に出る**機能ボタン**で再加工：
   - **短く／長く／フォーマル／カジュアル／箇条書き化**
   - **翻訳**（右のプルダウンで言語を選んで「翻訳」）
   - **元に戻す**（最初の清書へ戻す）
   - 再加工するたびに自動コピー＆（ONなら）Obsidian再保存

> 💡 連続でメモを取りたい時は、設定で **自動Obsidian保存をOFF** にすると、Obsidianへ切り替わらず
> アプリ内で録り続けられます。保存したい時だけ **「Obsidianに保存」** ボタンを押してください。

## 📒 Obsidian保存の仕組みと注意

- 保存は `obsidian://new?vault=…&file=フォルダ/ファイル名&content=本文` を起動して新規ノートを作る方式です
  （vault名・パス・本文はすべて `encodeURIComponent` でエンコード）。
- **前提**: Android版Obsidianがインストール済みで、**設定のvault名と保管庫名が完全一致**していること。
  一致しないと「保管庫が見つからない」エラーになります。
- **未インストール/起動失敗**時は、アプリ側で約1.5秒後に「Obsidianが開きません…」とトースト表示します。
- **長文の注意**: 本文が非常に長いとURLが極端に長くなります。Androidは比較的長いURLでも動きますが、
  端末・OSによって上限があり得ます。うまく開かない場合は、自動保存をOFFにして**コピー → Obsidianに手動貼り付け**で対応してください。
- ファイル名は設定テンプレ（`YYYY-MM-DD_HHmm` 等）で生成。秒トークンが無い場合は自動で秒を付け、
  同分内の重複を避けます（再加工で複数回保存しても別ノートになります）。

## 🗂️ （任意・上級）Googleドキュメントにも保存する — GAS

Obsidianとは別に、清書を**1つのGoogleドキュメントへ追記ログ**したい場合の設定です。不要ならスキップ。

1. https://script.google.com で **新しいプロジェクト**
2. エディタの内容を消し、`gas/Code.gs` の中身を**貼り付けて保存**
3. 右上 **デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**
   - 説明: 任意 / **実行ユーザー: 自分** / **アクセスできるユーザー: 全員**
   - 初回は権限承認（自分のアカウントを選び許可）
4. 発行された **ウェブアプリURL（末尾 `/exec`）** をコピー
5. アプリの設定 **「GAS Web App URL」** に貼り付け → 保存
6. 以後、清書のたびに Drive 内の **「AI音声メモ ログ」** ドキュメントへ日時付きで追記されます
   （ドキュメント名は `Code.gs` の `DOC_NAME` で変更可）。

> ※アプリは `no-cors` 送信のため、保存成否はアプリ側では確認できません。最初の数回はGoogleドキュメントに
> 追記されているか実際に確認してください。

## ⚠️ セキュリティ上の注意（必ずお読みください）

- この構成では **APIキーが端末内（localStorage）に保存され、ブラウザから直接 Anthropic API へ送信**されます（サーバー非経由）。
  - **共有端末・他人が触る端末では使わない**でください。
  - **利用上限・予算を絞った専用キー**を発行して使ってください（Anthropic Console でキーごとに上限設定可）。
  - キー漏洩時・端末を手放す時は、Console から**そのキーを revoke（失効）**してください。
- 音声認識（Web Speech API）は仕様上、Googleのサーバーで処理されることがあります。機微な内容にはご注意ください。

## 🔧 困ったときは

| 症状 | 対処 |
|---|---|
| マイクが反応しない | 状態テキストに出る `認識エラー: ○○` を確認。`not-allowed`→🔒からマイク許可 / `network`→通信確認 |
| 「APIエラー(401)」 | キーが誤り／失効。設定で再保存 |
| 「APIエラー(400)」でモデル名 | モデル欄を空にして既定 `claude-haiku-4-5` に戻す |
| Obsidianが開かない | Android版Obsidianを入れ、**vault名を完全一致**に。長文時はコピー→手動貼り付け |
| 自動コピーされない | 「コピー」ボタンで手動コピー |
| 音声認識が未対応と出る | **Android Chrome**で開く（iOS/一部ブラウザは非対応） |
| 変更が反映されない | PWAキャッシュのため、Chromeで再読込 or アプリ削除→再追加 |

## 📝 モデルについて

- 既定は **`claude-haiku-4-5`**（軽量・低コスト。清書・再加工・翻訳に最適）。設定でいつでも変更可。
- 最新モデル名は https://docs.claude.com で確認できます。
- 清書・各機能は `max_tokens: 2048` で実行します。
