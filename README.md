# 🎤 AI音声メモ（Android Chrome 専用 PWA）

ワンタップで録音 → Web Speech APIでテキスト化 → Claude（軽量モデル `claude-haiku-4-5`）が
読みやすい日本語に清書 → 画面表示＆**自動でクリップボードにコピー**。

ビルド不要・バックエンド不要。素のHTML/CSS/JSだけで動くので、**GitHub Pagesにファイルを
置くだけ**で公開できます。プログラミング知識がなくてもこの手順どおりで動きます。

---

## ✅ 事前に用意するもの

1. **GitHubアカウント**（無料）… https://github.com/signup
2. **Anthropic APIキー**… https://console.anthropic.com/ → 「API Keys」で発行

     必ず**利用上限／予算を絞った専用キー**を作ってください（後述の注意書き参照）。

## 🚀 メイン手順：GitHub Pages で公開する

### ステップ1. リポジトリを作る
1. https://github.com/new を開く
2. **Repository name** に `voice-memo` と入力
3. **Public** を選択（Pagesは公開リポジトリなら無料）
4. 緑の **Create repository** を押す

### ステップ2. ファイルをアップロードする
1. 作成されたリポジトリ画面で **Add file → Upload files** をクリック
2. この `voice-memo` フォルダの中身を**まるごとドラッグ＆ドロップ**
   - `index.html` などのファイルと、`icons` フォルダ（中の2つのpng）も一緒に
   - ※フォルダ構成（`icons/icon-192.png` の形）が保たれるようにアップロードしてください
3. 下の **Commit changes** を押す

### ステップ3. Pages を有効化する
1. リポジトリ上部の **Settings** を開く
2. 左メニューの **Pages** をクリック
3. **Build and deployment** の **Source** で **Deploy from a branch** を選択
4. **Branch** を **main** ／ フォルダを **/(root)** にして **Save**
5. 1〜2分待つと、上部に公開URLが表示されます
   例）`https://あなたのユーザー名.github.io/voice-memo/`

### ステップ4. スマホで開く
1. **Android の Chrome** で上記URLを開く
   （※ HTTPSで配信されるため、マイクとクリップボードが正しく動きます）
2. 右上の ⚙️（設定）を開き、**Anthropic APIキー**を貼り付けて **保存**
   - モデルは空欄でOK（既定 `claude-haiku-4-5`）。変更したい場合のみ入力。

### ステップ5. ホーム画面に追加（アプリ化）
- 初回アクセス時に画面下に **「ホーム画面に追加」** ボタンが出たらタップ
- 出ない場合は Chrome 右上 **⋮（メニュ）→ アプリをインストール／ホーム画面に追加**
- 追加後はホームのアイコンから**全画面アプリ**として起動できます

## 🟢 使い方

1. 中央の大きな丸ボタンをタップ → **録音開始**（赤く点滅）
2. 話す（連続録音。Androidで認識が一瞬切れても自動で再開します）
3. もう一度ボタンをタップ → **停止** → AIが清書 → 結果表示
4. 清書結果は**自動でクリップボードにコピー**され「コピー完了」と表示されます
   - 自動コピーが効かない端末では右下の **コピー** ボタンで手動コピーできます

## 🟣 代替手順：Vercel（ドラッグ＆ドロップで数分）

GitHubを使いたくない場合の最短ルート。

1. https://vercel.com/ にGitHub/メール等でサインアップ
2. ターミナルが使えるなら：
   ```
   npm i -g vercel
   cd voice-memo
   vercel
   ```
   表示される質問はEnterで進め、最後に表示されるURLにスマホでアクセス。
3. ターミナルを使わない場合は **Vercel → Add New → Project → Deploy** で
   `voice-memo` フォルダをアップロードするだけでもOK（自動でHTTPS公開されます）。

> Vercel／GitHub Pages どちらも **HTTPS** で配信されるため、マイク権限・
> `navigator.clipboard` の自動コピーが正しく動作します（HTTPでは動きません）。

## ⚠️ セキュリティ上の注意（必ずお読みください）

- この構成では **APIキーが端末内（localStorage）に保存され、ブラウザから直接 Anthropic
  API へ送信**されます。サーバーを介しません。
- そのため：
  - **共有端末・他人が触る端末では使わない**でください。
  - **利用上限・予算を絞った専用キー**を発行して使ってください
    （Anthropic Console でキーごとに上限設定が可能です）。
  - キーが漏れた・端末を手放す場合は、Console から**そのキーを失効（revoke）**してください。
- 音声認識（Web Speech API）は端末／ブラウザの仕様上、Googleのサーバーで処理されることが
  あります。機微な内容の取り扱いにはご注意ください。

## 🔧 困ったときは

| 症状 | 対処 |
|---|---|
| マイクが反応しない | アドレスバーの 🔒 → マイクを「許可」。HTTPSで開いているか確認 |
| 「APIエラー(401)」 | APIキーが誤り／失効。設定⚙️で正しいキーを再保存 |
| 「APIエラー(400)」でモデル名 | 設定のモデル欄を空にして既定 `claude-haiku-4-5` に戻す |
| 自動コピーされない | 右下の「コピー」ボタンで手動コピー（端末の権限仕様による） |
| 音声認識が未対応と出る | **Android Chrome** で開く（iOS／一部ブラウザは非対応） |
| 変更が反映されない | PWAキャッシュのため、Chromeで再読み込み or 一度アプリを削除→再追加 |

## 📝 モデルについて

- 既定は **`claude-haiku-4-5`**（軽量・低コスト。清書用途に最適）。
- 最新のモデル名は https://docs.claude.com で確認できます。設定画面でいつでも変更可能です。
- 清書は `max_tokens: 2048` で実行します（長文メモにも対応）。
